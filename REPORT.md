## 3) Что было исправлено (итерация 2 — 05.04.2026)

### Исправление 1: неверная связь predecessor (критично)

**Проблема:** Задача UID=3 («Регистрация бизнеса и расчётного счёта», WBS 1.1) содержала
`<PredecessorLink>` с `PredecessorUID=4` («Анализ ниши»). Логика перевёрнута: задача 3 стартует
27.03, задача 4 — 01.04. Predecessor был указан на более позднюю задачу.

**Исправление:**
- Удалён `PredecessorLink(PredecessorUID=4)` из задачи UID=3
- Добавлен `PredecessorLink(PredecessorUID=3, Type=1/FS)` к задаче UID=4

Корректная цепочка: Регистрация (1.1) → Анализ ниши (1.2) → Выбор ассортимента (1.3) → Сертификаты (1.4).

### Исправление 2: CalendarUID ресурсов (критично)

**Проблема:** Все 5 ресурсов (UID=2..6) ссылались на `CalendarUID=3` (календарь «Менеджер»),
а не на свои персональные календари.

**Исправление:**

| Ресурс UID | Имя | Было | Стало |
|-----------|-----|------|-------|
| 2 | Маркетолог | 3 | 4 |
| 3 | Юрист | 3 | 5 |
| 4 | Логист | 3 | 6 |
| 5 | Поставщик / производство | 3 | 7 |
| 6 | Контент-менеджер | 3 | 8 |

### Исправление 3: добавлен Calendar UID=8 для Контент-менеджера

**Проблема:** В секции `<Calendars>` присутствовали UID 1–7, но отсутствовал UID=8,
на который стал ссылаться ресурс «Контент-менеджер» после правки 2.

**Исправление:** Добавлен `<Calendar>` с UID=8, Name=«Контент-менеджер»,
BaseCalendarUID=1 (Стандартный), IsBaseCalendar=0.

## 5) Попытка создания .mpp — результат

**Установлено и проверено:**
- Java OpenJDK 21 — доступна ✓
- python-mpxj 16.1.0 + jpype1 1.6.0 — установлены ✓
- MPXJ успешно прочитал XML (26 задач, 7 ресурсов) ✓

**Попытка записи .mpp:**
- `org.mpxj.mpp.MppWriter` — класс **не существует** в MPXJ 16.1.0
- Доступные форматы записи в MPXJ: MPX, MSPDI/XML, JSON, Planner, Primavera XER/PMXML
- **MPP-writer в MPXJ отсутствует** — это задокументированное ограничение библиотеки

**Вывод:** `.mpp` не создан по объективной технической причине — ни один open-source
инструмент на Linux не умеет ЗАПИСЫВАТЬ бинарный `.mpp` формат (он проприетарный,
принадлежит Microsoft). MPXJ умеет только ЧИТАТЬ `.mpp`.

## 6) Как открыть XML и сохранить в .mpp вручную

### Вариант A — Microsoft Project (рекомендуется)

1. Откройте Microsoft Project.
2. **File → Open** → укажите `ЗАДАНИЕ_3_БАРАНОВА_ЭВЕЛИНА_ФИНАЛ.xml`.
3. При запросе импорта выберите открытие как MS Project XML.
4. Проверьте Gantt Chart, Resource Sheet, Task Usage — структура должна отображаться корректно.
5. **File → Save As** → тип файла **Project (*.mpp)**.
6. Сохраните как `ЗАДАНИЕ_3_БАРАНОВА_ЭВЕЛИНА_ФИНАЛ.mpp`.

### Вариант B — ProjectLibre

1. Скачайте ProjectLibre с сайта projectlibre.com (бесплатно).
2. **File → Open** → выберите `ЗАДАНИЕ_3_БАРАНОВА_ЭВЕЛИНА_ФИНАЛ.xml`.
3. Проверьте корректность отображения задач / ресурсов / связей.
4. Сохраните в формате ProjectLibre (.pod) или используйте для проверки.
5. Для именно `.mpp` используйте Microsoft Project (вариант A).
